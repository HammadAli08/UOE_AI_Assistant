ABSTRACT
================================================================================

This project presents an AI-Powered Academic Assistant designed specifically for
the University of Education, Lahore. The system enables students to ask questions
in both English and Roman Urdu, receiving accurate, source-grounded answers from
official university documents.

The project implements a Retrieval-Augmented Generation (RAG) pipeline with three
knowledge bases populated from official university documents. The key innovation
is the Self-Correcting Smart RAG system that grades retrieved chunks, rewrites
queries when necessary, and retries up to three times (four retrieval attempts
total) before returning the best available result.

The backend is built using FastAPI with Server-Sent Events (SSE) streaming for
real-time response delivery. The frontend features a modern dark glassmorphic
design with smooth animations using React and Framer Motion. The system maintains
conversational memory using Redis, storing up to ten turns of context with
automatic expiration.

The knowledge base consists of three namespaces: BS/ADP Programs, MS/PhD
Programs, and University Rules and Regulations. Each namespace has specialized
prompts optimized for generating accurate responses specific to that domain.

Keywords: Artificial Intelligence, Academic Assistant, Retrieval-Augmented
Generation, University Automation, Smart RAG, Pinecone, FastAPI, React

1.1 INTRODUCTION
--------------------------------------------------------------------------------

The University of Education, Lahore serves thousands of students across multiple
campuses. Students often struggle to find accurate information about course
structures, admission requirements, examination policies, and university
regulations. The existing information retrieval systems are either outdated or
inefficient, requiring students to manually search through lengthy PDF documents.

This project aims to solve this problem by developing an AI-Powered Academic
Assistant that can understand student queries in natural language and provide
accurate, source-grounded answers from official university documents. The system
supports queries in both English and Roman Urdu, making it accessible to a wider
audience of Pakistani students.

The proposed system utilizes modern Artificial Intelligence techniques,
specifically Retrieval-Augmented Generation (RAG), to combine the power of large
language models with the accuracy of document-based retrieval. By leveraging
vector embeddings and semantic search, the system can understand the intent
behind user queries and retrieve the most relevant information from the
university's knowledge base.

1.2 GENERAL OVERVIEW OF THE SYSTEM
--------------------------------------------------------------------------------

The UOE AI Assistant is a full-stack web application that consists of three main
components:

Frontend Application:
The frontend is a Single Page Application (SPA) built using React.js with Vite
as the build tool. It features a modern dark-themed interface with glassmorphic
design elements. The main features include:
- Landing page with project information
- Interactive chat interface with streaming responses
- Namespace selector for different knowledge bases
- User authentication via Supabase
- Chat history persistence
- Source citation metadata attachment in assistant responses
- Feedback mechanism for quality improvement

Backend API:
The backend is built using FastAPI, a modern Python web framework. It provides
RESTful endpoints and supports Server-Sent Events (SSE) for real-time streaming.
The backend handles:
- Query processing and enhancement
- Vector database retrieval
- LLM response generation
- Conversational memory management
- Namespace discovery for frontend selection
- Feedback logging

Data Infrastructure:
The system uses multiple external services:
- Pinecone: Vector database for semantic search
- Redis Cloud: Session memory (TTL-based context storage)
- OpenAI: GPT-4o-mini for text generation and text-embedding-3-large for
  embeddings
- Supabase: Authentication and chat persistence
- LangSmith: Observability and tracing

1.3 PROBLEM STATEMENT
--------------------------------------------------------------------------------

Students at the University of Education, Lahore face several challenges when
trying to access academic information:

1. Information Scattered Across Documents: Course outlines, admission criteria,
   fee structures, and regulations are stored in separate PDF files, making it
   difficult for students to find specific information.

2. Outdated Search Methods: The current systems rely on keyword matching, which
   fails to understand the semantic intent behind user queries.

3. Language Barrier: Many students prefer to ask questions in Roman Urdu (Urdu
   written in Latin script), but existing systems only support English.

4. Lack of Conversational Context: Students cannot have follow-up conversations
   about their queries, requiring them to rephrase questions repeatedly.

5. No Source Attribution: Existing systems do not provide citations, making it
   difficult for students to verify the information.

6. Poor User Experience: The current interfaces are not user-friendly and lack
   modern interaction patterns.

2.2 RETRIEVAL-AUGMENTED GENERATION
--------------------------------------------------------------------------------

Retrieval-Augmented Generation (RAG) is a technique that combines the strengths
of information retrieval systems with large language models. Instead of relying
solely on the model's internal knowledge, RAG systems first retrieve relevant
documents from an external knowledge base and then use these documents to
generate more accurate and grounded responses.

Key benefits of RAG include:
- Reduced hallucination (model generating false information)
- Ability to access up-to-date information
- Source attribution for generated content
- Reduced computational costs compared to fine-tuning

2.3 SMART RAG AND SELF-CORRECTING RETRIEVAL
--------------------------------------------------------------------------------

Traditional RAG systems have a limitation: they perform a single retrieval pass
and generate responses based on whatever documents are returned. If the initial
retrieval fails to find relevant documents, the final response will be poor
quality.

Smart RAG addresses this by implementing a self-correcting loop:
1. Initial retrieval of documents
2. LLM-based grading of retrieved chunks for relevance
3. If insufficient relevant chunks, rewrite query and retry
4. Repeat up to maximum retries
5. Generate answer using all relevant chunks collected

This approach significantly improves retrieval quality, especially for complex
or ambiguous queries.

2.4 VECTOR DATABASES AND SEMANTIC SEARCH
--------------------------------------------------------------------------------

Vector databases like Pinecone store document embeddings - numerical
representations of text that capture semantic meaning. Unlike keyword-based
search, semantic search can understand that "admission requirements" and
"enrollment criteria" are related concepts, even if they don't share exact
words.

Key concepts:
- Embeddings: Dense numerical vectors representing text meaning
- Cosine similarity: Measure of semantic closeness between vectors
- Namespaces: Separate partitions within a vector database for different
  document collections

2.5 CONVERSATIONAL MEMORY IN AI SYSTEMS
--------------------------------------------------------------------------------

Conversational memory allows AI systems to maintain context across multiple
turns of dialogue. This is essential for natural conversations where users can
ask follow-up questions without repeating context.

Implementation approaches:
- Short-term memory: Redis or in-memory storage for session-based context
- Long-term memory: Persistent storage for user preferences and chat history
- Sliding window: Limiting memory to recent N turns to manage context length

3.3 DATA FLOW
--------------------------------------------------------------------------------

Standard RAG Flow:
User Query -> Query Enhancement -> Vector Retrieval (Top-5) -> LLM Generation
-> Streamed Answer

Smart RAG Flow:
User Query
   |
   -> Query Enhancement (GPT-4o-mini rewrites for optimal retrieval)
   |
   -> Vector Retrieval (Pinecone, 5 docs, 3072-dim embeddings)
   |
   -> Chunk Grading (GPT-4o-mini scores each chunk as relevant/irrelevant)
   |
   -> If >= 2 relevant chunks -> Generate answer
   -> If < 2 relevant -> Rewrite query -> Re-retrieve -> Re-grade
                       |
                       -> Retry up to 3 times with progressive strategy
                               |
                               -> Found enough -> Generate answer
                               -> Some found -> Best-effort answer
                               -> Zero found -> Clarification / Fallback

4.1 BACKEND IMPLEMENTATION
--------------------------------------------------------------------------------

The backend is implemented in Python using FastAPI, providing high performance
and native support for asynchronous operations.

Key Files:

- rag_pipeline/query_enhancer.py: Query rewriting
- rag_pipeline/memory.py: Redis conversation memory
- rag_pipeline/smart_rag/: Self-correcting retrieval components

4.2 FRONTEND IMPLEMENTATION
--------------------------------------------------------------------------------

The frontend is a React application built with Vite, featuring a modern
glassmorphic design.

Key Files:
- src/App.jsx: Main application with routing
- src/components/Chat/: Chat interface components
- src/components/Landing/: Landing page components
- src/components/Auth/: Authentication components
- src/hooks/useChat.js: SSE streaming chat logic
- src/store/: Zustand state management

4.3 RAG PIPELINE COMPONENTS
--------------------------------------------------------------------------------

Query Enhancer:
Rewrites user queries to improve retrieval performance. Handles:
- Roman Urdu to English conversion
- Query expansion with synonyms
- Ambiguity resolution

Retriever:
Performs semantic search using Pinecone with ensemble retrieval:
- Dense retrieval (vector similarity)
- Sparse retrieval (BM25 keyword matching)
- Reciprocal Rank Fusion for combining results

Smart RAG:
Implements self-correcting retrieval:
- Grader: Evaluates chunk relevance using LLM
- Rewriter: Reformulates queries for better results
- Processor: Orchestrates retry logic

Generator:
Generates responses using retrieved documents:
- Streaming token-by-token output
- Source citation inclusion
- Chat history consideration
